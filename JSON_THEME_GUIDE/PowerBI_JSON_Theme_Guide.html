SECTION 01

What is a JSON Theme?

A single configuration file that controls the entire visual appearance of your Power BI report.

📄
JSON — JavaScript Object Notation
A text-based key-value data format

A Power BI JSON theme file defines colors, fonts, borders, backgrounds, and chart defaults for every visual type — in one centralized file. Applying it is a single action that formats everything instantly.

JSON — Basic Structure
{
  "name": "Anshul_RA1_SalesAnalytics",     ← Theme identifier
  "dataColors": ["#4FACFE", "#00D4B4"],  ← Chart colors
  "background": "#1E2130",              ← Page background
  "foreground": "#FFFFFF",              ← Default text
  "visualStyles": {                    ← Per-visual settings
    "card": { ... },                    ← Only cards
    "slicer": { ... },                  ← Only slicers
    "lineChart": { ... }                ← Only line charts
  }
}
❌ Without JSON Theme (Manual)
Select Card 1 → Format → Background → set color
Select Card 2 → Format → Background → same color
Select Card 3 → Format → Background → same color
Select Line Chart → Format → Background → same color
Select Bar Chart → Format → Background → same color
50+ steps per page × 5 pages = 250+ manual steps!
Human errors → Inconsistent colors
✅ With JSON Theme
Create theme.json file (one time!)
View → Themes → Browse → Upload
DONE! All visuals formatted instantly!
All 5 pages consistent automatically
No human error possible
Takes 30 seconds total!

SECTION 03

JSON Theme Structure

Every property explained — from top level to visual-specific settings.

🏗️
Complete Structure Overview
JSON — Full Structure
{
  "name": "ThemeName",          ← Appears in Power BI themes list

  "dataColors": [             ← Array of chart/data colors
    "#4FACFE",               ← Color 1 (first data series)
    "#00D4B4",               ← Color 2 (second data series)
    "#68D391"                ← Color 3 etc...
  ],

  "background": "#1E2130",    ← Default page background
  "foreground": "#FFFFFF",    ← Default text color
  "tableAccent": "#4FACFE",   ← Table/matrix highlights

  "visualStyles": {           ← Per-visual type formatting

    "*": {                      ← Applies to ALL visuals
      "*": {                    ← All states
        "fontFamily": [...],
        "fontSize": [...]
      }
    },

    "card": {                  ← Only Card visuals
      "*": { "background": [...] },
      "labels": { ... },        ← Big number formatting
      "categoryLabels": { ... } ← Label text formatting
    },

    "slicer": { ... },         ← Only Slicer visuals
    "lineChart": { ... },      ← Only Line Charts
    "barChart": { ... }        ← Only Bar Charts
  },

  "page": {                    ← Page-level settings
    "background": {            ← Canvas area color
      "color": "#1E2130",
      "transparency": 0
    },
    "outspace": {              ← Outer area color
      "color": "#141820",
      "transparency": 0
    }
  }
}
SECTION 04

Complete Parameter Reference

Every property type you will use — with exact JSON syntax.

🎨
Background Property
Sets background color of any visual
JSON
"background": [{
  "color": {
    "solid": {
      "color": "#252B3B"  ← Your hex color here
    }
  }
}]
🔲
Border Property
Visual border with color and rounded corners
JSON
"border": [{
  "color": { "solid": { "color": "#4FACFE" } },
  "show": true,    ← true = show border, false = hide
  "radius": 8      ← Corner roundness (0=square, 16=round)
}]
💡Radius values: 0 = sharp corners, 4 = subtle, 8 = modern (our theme), 16 = very round (pill shape)
📝
Font Properties
Color, size, and family
JSON
// Font Color
"fontColor": [{ "solid": { "color": "#FFFFFF" } }]

// Font Size (in points)
"fontSize": [{ "value": 13 }]

// Font Family
"fontFamily": [{ "value": "Segoe UI Semibold" }]

// Common font options:
// "Segoe UI"          ← Standard
// "Segoe UI Semibold" ← Bold headers
// "Segoe UI Light"    ← Subtle labels
// "DIN"               ← Clean modern
// "Calibri"           ← Traditional
📊
dataColors — Chart Color Array
How Power BI assigns colors to data series
JSON
"dataColors": [
  "#4FACFE",   ← Index 0 → First series (most important!)
  "#00D4B4",   ← Index 1 → Second series
  "#68D391",   ← Index 2 → Third series
  "#F0B429",   ← Index 3 → Fourth series
  "#FC8181",   ← Index 4 → Fifth series
  "#B794F4"    ← Index 5+ → Additional series
]

// Example — Line chart with 4 years:
// 2014 line → #4FACFE (Blue)
// 2015 line → #00D4B4 (Teal)
// 2016 line → #68D391 (Green)
// 2017 line → #F0B429 (Gold)
💡Always put your most important/primary brand color at index 0 — it will appear on the most prominent data series first!
⚡
Wildcard "*" — Global Defaults
Apply settings to every visual at once
JSON
"visualStyles": {
  "*": {                  ← "*" = ALL visual types
    "*": {                ← "*" = ALL states
      "fontFamily": [{ "value": "Segoe UI" }],
      "fontSize": [{ "value": 11 }],
      "background": [{
        "color": { "solid": { "color": "#252B3B" } }
      }]
    }
  },

  // This overrides the global default for cards only:
  "card": {
    "labels": {
      "fontSize": [{ "value": 28 }]  ← Cards get 28pt, others get 11pt
    }
  }
}

// Priority: Manual override > Visual specific > Wildcard "*"

Visual-Specific Formatting

Sub-sections for each visual type — what each property controls.

🃏
Card Visual — Sub-sections
┌──────────────────────────┐ │ │ │ $2.3M │ ← "labels" → color, fontSize, fontFamily │ Total Sales │ ← "categoryLabels" → color, fontSize │ │ └──────────────────────────┘ ↑ Background color → "*" → "background" ↑ Border color/radius → "*" → "border"
JSON — Card
"card": {
  "*": {
    "background": [{ "color": { "solid": { "color": "#252B3B" } } }],
    "border": [{ "color": { "solid": { "color": "#4FACFE" } }, "show": true, "radius": 8 }]
  },
  "labels": {                         ← The BIG number ($2.3M)
    "color": [{ "solid": { "color": "#4FACFE" } }],
    "fontSize": [{ "value": 28 }],
    "fontFamily": [{ "value": "Segoe UI Semibold" }]
  },
  "categoryLabels": {                 ← "Total Sales" text below
    "color": [{ "solid": { "color": "#A0AEC0" } }],
    "fontSize": [{ "value": 11 }]
  }
}
🔽
Slicer Visual — Sub-sections
┌────────────────┐ │ Year │ ← "header" → fontColor, fontSize, background │ All [▼] │ ← "items" → fontColor, fontSize, background └────────────────┘ ↑ Outer border → "*" → "border"
JSON — Slicer
"slicer": {
  "*": {
    "background": [{ "color": { "solid": { "color": "#252B3B" } } }],
    "border": [{ "color": { "solid": { "color": "#4FACFE" } }, "show": true, "radius": 6 }]
  },
  "header": {                       ← "Year" / "Region" label
    "fontColor": [{ "solid": { "color": "#FFFFFF" } }],
    "fontSize": [{ "value": 11 }],
    "fontFamily": [{ "value": "Segoe UI Semibold" }]
  },
  "items": {                        ← Dropdown option values
    "fontColor": [{ "solid": { "color": "#FFFFFF" } }],
    "fontSize": [{ "value": 11 }]
  }
}
📈
Line Chart — Sub-sections
Monthly Sales Trend ← "title" → fontColor, fontSize ┌────────────────────────┐ $300K ─ ────────────── │ ← "valueAxis" → gridlineColor, labelColor $200K ─ │ $100K ─ │ Jan Feb Mar Apr │ ← "categoryAxis" → labelColor └────────────────────────┘ ■ 2016 ■ 2017 ← "legend" → labelColor
🌊
Waterfall Chart — Sentiment Colors
JSON — Waterfall
"waterfallChart": {
  "sentimentColors": {
    "totalBar": "#4FACFE",      ← Starting/ending total bar
    "increasedBar": "#68D391", ← Positive growth bar (Green)
    "decreasedBar": "#FC8181"  ← Negative/decline bar (Red)
  }
}
💡Sentiment colors make waterfall charts intuitive — green for growth, red for decline. No need for legends!
📊
Matrix Visual — Alternating Rows (Zebra Striping)
JSON — Matrix
"matrix": {
  "header": {
    "fontColor": [{ "solid": { "color": "#FFFFFF" } }],
    "background": [{ "color": { "solid": { "color": "#1E2130" } } }]
  },
  "values": {
    "fontColorPrimary": [{ "solid": { "color": "#FFFFFF" } }],
    "backgroundColorPrimary": [{ "color": { "solid": { "color": "#252B3B" } } }],
    "fontColorSecondary": [{ "solid": { "color": "#A0AEC0" } }],      ← Even rows
    "backgroundColorSecondary": [{ "color": { "solid": { "color": "#1E2A3B" } } }] ← Even rows
  },
  "totals": {
    "fontColor": [{ "solid": { "color": "#4FACFE" } }],
    "background": [{ "color": { "solid": { "color": "#1E2130" } } }]
  }
}

SECTION 07

How to Apply a Theme

Step by step — from saving the file to seeing results in Power BI.

1
Save the JSON file
Save as Anshul_RA1_Theme.json in your project folder: 01_Sales_Analytics/templates/
2
Open View tab in Power BI Desktop
Top ribbon → View tab → Find the "Themes" section on the left side
3
Browse for themes
Click the dropdown arrow next to the theme thumbnails → "Browse for themes" → Navigate to your .json file → Open
4
Theme applies instantly!
All visuals across all pages update immediately. Check cards, slicers, charts — all should reflect the new colors and fonts.
5
Fine-tune individual visuals if needed
Select any visual → Format pane → Override specific settings. This only affects that one visual — the theme file is unchanged. Manual overrides always take priority over the theme.
⚠️Important: Save your JSON theme file in the templates/ folder inside your project and commit it to GitHub. This way anyone cloning your repo can apply the same theme — essential for team projects!
SECTION 08

Common Mistakes to Avoid

Learn from these before you encounter them.

❌ Too Many Colors
Using 15 different colors in dataColors makes charts look chaotic. Stick to 5-6 primary colors max.
❌ Low Contrast Text
Dark background + dark text = unreadable. Always check contrast ratio — minimum 4.5:1 for readability.
❌ Inconsistent Radius
Some visuals rounded, some sharp looks messy. Use same radius value everywhere in the theme.
❌ Not Testing Preview Mode
Edit mode shows handles and borders. Press Ctrl+Shift+F11 to preview the actual end-user view.
❌ Forgetting Page Background
Setting visual backgrounds without setting page background leaves white canvas. Always set both "background" and "outspace" in the page section.
❌ Not Saving to Git
Theme file not committed means team cannot reproduce your design. Always commit theme.json to the templates/ folder.
SECTION 09

Interview Questions

Common questions about Power BI themes — with strong answers.

Q
What is a Power BI JSON theme file and why would you use one?
A JSON theme file is a centralized configuration file that defines the visual appearance of an entire Power BI report — colors, fonts, borders, backgrounds, and chart defaults for every visual type. I use it because it ensures 100% design consistency across all pages and visuals, enforces brand guidelines automatically, and makes maintenance trivial. If a brand color needs to change, updating one line in the JSON file instantly updates every visual in the report rather than requiring 50+ manual changes.
Q
How does the wildcard "*" in visualStyles work?
The wildcard "*" in visualStyles applies settings to every visual type when used as the key. A nested "*" applies to all states of that visual. It acts as a global default — the lowest priority in the cascade. More specific rules like "card" override the wildcard for card visuals only. Individual manual formatting on a specific visual always takes the highest priority and overrides both the wildcard and visual-specific theme settings.
Q
What is dataColors and why does the order of colors matter?
dataColors is an array of hex color codes that Power BI cycles through when displaying multiple data series in charts. The order matters because the first color at index 0 goes to the first data series — the most prominent one. I always place the primary brand color at index 0 so it appears on the most important data. For example, in a line chart showing 4 years, index 0 gets the first year and index 3 gets the last year.
Q
How do you apply a JSON theme in Power BI Desktop?
In Power BI Desktop, go to the View tab, click the dropdown arrow in the Themes section, select "Browse for themes", navigate to the JSON file, and open it. The theme applies instantly across all pages. Individual visuals can still be manually overridden after application — manual formatting always takes priority over the theme settings.
Q
What is the difference between background and outspace in the page section?
Background sets the canvas area color — the actual report page where visuals are placed. Outspace sets the color of the area outside the canvas boundary — the grey border region visible in edit mode. Setting both creates a cohesive dark environment during development. In presentation mode, only the background color is visible to end users.
Q
How would you handle a client request to change the primary brand color across an existing report?
If the report was built with a JSON theme, it is a two-minute change — open the JSON file, find and replace the old color hex code with the new one, save, and re-upload the theme. All visuals update instantly with zero risk of missing any. This is exactly why I always use JSON themes in production reports — design changes that would take hours manually become trivial with a centralized theme file.
